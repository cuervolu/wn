## Instalación

La forma recomendada hoy es instalar el binario oficial con los scripts de release. Si prefieres
compilarlo tú mismo, eso queda como camino alternativo.

<Tabs items={['macOS / Linux', 'Windows', 'Desde la fuente']}>
  <Tab value="macOS / Linux">

  ```sh
  curl --proto '=https' --tlsv1.2 -LsSf https://github.com/cuervolu/wn/releases/latest/download/wn-cli-installer.sh | sh
  ```

  </Tab>
  <Tab value="Windows">

  ```powershell
  powershell -ExecutionPolicy Bypass -c "irm https://github.com/cuervolu/wn/releases/latest/download/wn-cli-installer.ps1 | iex"
  ```

  </Tab>
  <Tab value="Desde la fuente">

  ```sh
  git clone https://github.com/cuervolu/wn
  cd wn
  cargo build --release
  ./target/release/wn
  ```

  </Tab>
</Tabs>

<Tabs items={['Verificar', 'Actualizar']}>
  <Tab value="Verificar">

  ```sh
  wn --version
  ```

  </Tab>
  <Tab value="Actualizar">

  ```sh
  wn update
  ```

  </Tab>
</Tabs>

## Tu primer `.cl`

<Steps>
  <Step>

  ### Crea el archivo

  Guarda esto como `hola.cl`:

  ```wn
  pega saludar(nombre) {
    "wena, " + nombre + ", wn!"
  }

  lorea(saludar("mundo"))
  ```

  </Step>
  <Step>

  ### Ejecútalo

  ```sh
  wn hola.cl
  ```

  </Step>
  <Step>

  ### Métete al REPL si quieres webear más

  Si corres `wn` sin archivo, entras al REPL interactivo. Para salir, dile `chao`.

  </Step>
</Steps>

## Sintaxis en 30 segundos

Las variables mutables se declaran con `wea`, las constantes con `duro`, y las funciones con
`pega`. No necesitai punto y coma obligatorio, y la última expresión de una función puede devolver
sola.

```wn
wea n = 42
duro pi = 3.14
wea xs = [1, 2, 3]
wea m = { lenguaje: "WN++" }

pega saludar(nombre) {
  "wena, " + nombre + ", wn!"
}

lorea(cachar(xs))
lorea(largo(xs))
lorea(saludar("Cuervo"))
```

## Palabras clave

<Tabs items={['Base', 'Control de flujo']}>
  <Tab value="Base">

  | Chileno | Equivale a | Pa' qué sirve |
  | --- | --- | --- |
  | `wea` | `let` | Declara una variable mutable |
  | `duro` | `const` | Declara una constante |
  | `pega` | `fn` | Define una función |
  | `devolver` | `return` | Devuelve antes de tiempo |
  | `ojo / cago` | `try / catch` | Maneja errores |

  </Tab>
  <Tab value="Control de flujo">

  | Chileno | Equivale a | Pa' qué sirve |
  | --- | --- | --- |
  | `cachai` | `if` | Condicional |
  | `si no` | `else` | La otra rama |
  | `mientras` | `while` | Repite mientras se cumpla |
  | `para ... en` | `for in` | Itera sobre colecciones |
  | `cortala / sigue` | `break / continue` | Controla loops |

  </Tab>
</Tabs>

<Cards>
  <Card title="wn-core">
    El corazón del lenguaje: lexer, parser, AST y semántica base.
  </Card>
  <Card title="wn-diagnostics">
    La capa de errores y reportes para que los mensajes del lenguaje salgan bien armados.
  </Card>
  <Card title="wn-stdlib">
    El puente entre el runtime y la librería estándar que usa la CLI y el VM.
  </Card>
  <Card title="wn-vm">
    Compilador a bytecode, stack VM, benches y GC mark-and-sweep con `collect_garbage()`.
  </Card>
  <Card title="wn-cli">
    La herramienta `wn`: ejecuta archivos, abre el REPL, desensambla bytecode y exporta AST.
  </Card>
  <Card title="wn-wasm">
    La capa para correr WN++ desde WebAssembly y conectar el runtime con superficies web.
  </Card>
</Cards>

leer el pipeline completo y ver cómo cambia el modelo de ejecución sin salir del mismo proyecto.
